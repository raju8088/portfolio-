Track 1 — Python Core
From zero to confident Python developer
PHASE 1.1
Python Basics
Variables, types, operators, conditionals, loops — the skeleton of every Python program. Master the interpreter and basic I/O.
Variables & Types if / elif / else for / while loops Input / Output Type Casting String Methods
Weeks 1–2
PHASE 1.2
Data Structures (Built-in)
Deeply understand Python's built-in containers. Know when to use each one and their time/space implications.
Lists Tuples Sets Dicts List Comprehension Dict Comprehension Nested Structures
Weeks 3–4
PHASE 1.3
Functions & Scope
Functions are first-class in Python. Learn args, kwargs, closures, decorators, and lambda for writing clean, reusable code.
def / return *args / **kwargs Lambda Closures Decorators map / filter / zip Recursion
Weeks 5–6
PHASE 1.4
Advanced Python
File I/O, exception handling, modules, generators, iterators, and context managers. Write Python that works in the real world.
File I/O try / except / finally Generators Iterators Context Managers Modules & Packages Virtual Environments
Weeks 7–8
🧩
Track 2 — Object-Oriented Programming
Model the real world in code with classes and design patterns
PHASE 2.1
Classes & Objects
Understand how classes are blueprints and objects are instances. Master constructors, instance vs class attributes, and methods.
class keyword __init__ self Instance Methods Class Methods Static Methods __str__ / __repr__
Weeks 9–10
PHASE 2.2
4 Pillars of OOP
Encapsulation, Abstraction, Inheritance, and Polymorphism. The core concepts every software engineer must internalize deeply.
Encapsulation Abstraction Inheritance Polymorphism Method Overriding super() MRO
Weeks 11–12
PHASE 2.3
Magic Methods & Properties
Python's dunder methods make your objects behave like built-ins. Properties enable controlled attribute access.
__add__ / __len__ __eq__ / __lt__ __iter__ / __next__ @property Getter / Setter __slots__ Operator Overloading
Weeks 13–14
PHASE 2.4
Design Patterns & SOLID
Learn the SOLID principles and common design patterns. This is how professional codebases are structured — and what interviews often test.
SOLID Principles Singleton Factory Observer Strategy Decorator Pattern Abstract Classes (ABC)
Weeks 15–16
⚡
Track 3 — Data Structures & Algorithms
Crack coding interviews and write efficient programs
PHASE 3.1
Complexity & Arrays
Before writing any algorithm, understand how to measure it. Big O notation, time vs space tradeoffs, and array-based problem patterns.
Big O Notation Time Complexity Space Complexity Arrays & Strings Two Pointers Sliding Window Prefix Sum
Weeks 17–18
PHASE 3.2
Linked Lists, Stacks & Queues
Pointer-based structures that power operating systems and language runtimes. Implement from scratch in Python using classes.
Singly Linked List Doubly Linked List Stack (LIFO) Queue (FIFO) Deque collections.deque Fast & Slow Pointer
Weeks 19–20
PHASE 3.3
Trees & Graphs
Non-linear structures that model hierarchies and networks. BST, heaps, BFS, DFS, and Dijkstra's algorithm — heavily tested in interviews.
Binary Tree BST Tree Traversals Heap / Priority Queue Graph (adj. list) BFS / DFS Dijkstra's Topological Sort
Weeks 21–23
PHASE 3.4
Sorting, Searching & Hashing
Master the classics. Every sorting algorithm, binary search variations, and hash map patterns. Foundation for 80% of LeetCode problems.
Bubble / Selection Insertion Sort Merge Sort Quick Sort Binary Search Hash Maps Hash Sets Counting Sort
Weeks 24–25
PHASE 3.5
Recursion & Dynamic Programming
The hardest and most rewarding topic in DSA. Start with recursion, move to memoization, then bottom-up DP. Solve classic DP problems pattern by pattern.
Recursion Patterns Backtracking Memoization Tabulation Knapsack Longest Common Subseq. Coin Change DP on Grids
Weeks 26–28
PHASE 3.6
Greedy & Tries
Greedy algorithms make locally optimal choices. Tries are essential for autocomplete and IP routing. Both appear frequently in advanced interviews.
Greedy Strategy Interval Problems Activity Selection Huffman Coding Trie (Prefix Tree) Insert / Search / Delete Word Search
Weeks 29–30
🚀
Track 4 — Pro Programming
Write production-grade code and ace system design
PHASE 4.1
Testing & Clean Code
Professional developers test their code. Learn unit testing, TDD, and how to write readable, maintainable Python that others can work with.
pytest unittest Mocking TDD Cycle PEP 8 Type Hints Docstrings
Week 31
PHASE 4.2
System Design Basics
Understand how large-scale systems are designed. Databases, caching, load balancing, APIs — the concepts that separate engineers from programmers.
Scalability SQL vs NoSQL REST APIs Caching (Redis) Load Balancing Message Queues CAP Theorem
Week 32
